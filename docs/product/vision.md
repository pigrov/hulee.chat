# Vision

Hulee - платформа для коммуникаций компании с клиентами и сотрудниками. Продукт объединяет inbox, клиентские диалоги, внутренний мессенджер, поддержку, телефонию, внешние каналы, AI-сервисы, публичные API и интеграции с корпоративными системами.

## Главная цель

Создать один продукт, который может работать:

- как online/SaaS-сервис на инфраструктуре Hulee;
- как on-prem/private deployment на инфраструктуре клиента;
- как enterprise SaaS с выделенной инфраструктурой для крупной компании.

При этом core должен оставаться единым, а отличия компаний должны выражаться через конфигурацию, лицензии, модули, workflow и company-layer.

## Для кого

- Компании с распределенными командами продаж, сервиса, поддержки и сопровождения.
- Компании, которым нужен единый inbox по клиентам из мессенджеров, соцсетей, телефонии и публичных API.
- Компании, которым нужен внутренний мессенджер и поддержка сотрудников.
- Enterprise-клиенты, которым нужны on-prem, SSO, аудит, контроль данных и кастомные интеграции.

## Продуктовые возможности

- Единая карточка клиента.
- Единый inbox по клиентам, группам, обращениям и внутренним диалогам.
- Внутренний мессенджер для сотрудников.
- Поддержка сотрудников по направлениям продукта.
- Подключаемые каналы: Telegram, VK, MAX, WhatsApp, SMS и другие.
- Подключаемая телефония: MegaPBX, Билайн, Mango, UIS и другие.
- Подключаемая авторизация: email/password, Bitrix24, ADFS, LDAP, OIDC/SAML.
- Публичный API для регистрации клиентов, отправки сообщений и событий.
- Webhooks и outbound-интеграции.
- AI-модули: транскрибация, summary, классификация, аналитика качества.
- SaaS subscription/billing и on-prem license model.

## Принцип продукта

Core должен владеть бизнес-сущностями и жизненным циклом данных. Интеграции должны владеть только взаимодействием с внешним миром. Company-layer должен расширять продукт, а не ломать core.
